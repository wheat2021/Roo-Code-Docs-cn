## 概述

Roo Code 市场是一个发现和安装社区贡献扩展（称为 MCP 和模式）的中心枢纽。它使您能够轻松扩展 Roo Code 的功能，以适应您特定的需求和工作流程。

### 主要特性
- **可发现性**: 浏览精选的 MCP 和模式列表。
- **简易安装**: 单击即可安装和移除项目。
- **社区驱动**: 访问来自 Roo Code 社区不断增长的扩展集合。
- **项目与全局范围**: 为特定项目或所有项目安装项目。

---

## 入门

Roo Code 市场可直接在 VS Code 的 Roo Code 扩展中使用。通过单击顶部菜单栏中的市场图标 <Codicon name="extensions" /> 来访问它。

---



## 市场项目

市场提供两种类型的项目：

### MCP (模型上下文协议)
MCP 允许 Roo Code 连接并与各种 AI 模型、API 和其他外部工具进行交互。通过安装 MCP，您可以配置 Roo Code 以使用不同的语言模型（如来自 OpenAI、Anthropic 或其他公司的模型）或与其他服务集成。了解更多关于[什么是 MCP？](/features/mcp/what-is-mcp)以及[如何在 Roo Code 中使用 MCP](/features/mcp/using-mcp-in-roo)。

<img src="/img/marketplace/marketplace-1.png" alt="MCP 标签页显示可用的 MCP，带有安装和移除按钮" width="500" />

### 模式
模式是定制的指令和规则集，可为特定任务量身定制 Roo Code 的行为。例如，您可能会找到一个为创建 React 组件而优化的“React 组件”模式，或一个用于编写技术文档的“文档编写者”模式。了解更多关于[使用模式](/basic-usage/using-modes)和[创建自定义模式](/features/custom-modes)。

<img src="/img/marketplace/marketplace-4.png" alt="模式标签页显示可用的模式" width="500" />

## 安装范围

当您从市场安装项目时，可以选择在项目级别或全局级别进行安装。

### 项目级安装
- **范围**: 该项目仅在当前的 VS Code 工作区（您的项目）内可用。
- **配置文件**:
    - **MCPs**: 项目根目录下的 [`.roo/mcp.json`](/features/mcp/using-mcp-in-roo#configuring-mcp-servers)。
    - **模式**: 项目根目录下的 [`.roomodes`](/features/custom-modes#3-manual-configuration-yaml--json)。
- **使用场景**: 当某个项目特定于特定项目的需求，或者您想与团队共享项目特定的配置时，这非常有用。

### 全局安装
- **范围**: 该项目在您所有的 VS Code 工作区中都可用。
- **配置文件**:
    - **MCPs**: Roo Code 扩展全局设置目录中的 [`mcp_settings.json`](/features/mcp/using-mcp-in-roo#configuring-mcp-servers)。
    - **模式**: Roo Code 扩展全局设置目录中的 [`custom_modes.yaml`](/features/custom-modes)。
- **使用场景**: 这对于您希望在所有项目中使用的项目是理想的，例如最喜欢的模式或常用的 MCP。

## 使用市场

### 浏览和筛选

您可以在市场视图中浏览所有可用的项目。要查找特定项目：
- **搜索**: 使用搜索栏按名称或描述查找项目。
- **按类型筛选**: 仅显示 MCP 或仅显示模式。
- **按标签筛选**: 查找与特定技术或任务相关的项目。

### 安装项目

1. 找到您想要安装的项目。
2. 点击“安装”按钮。
3. 选择是为当前**项目**安装还是**全局**安装。

#### 安装 MCP
<img src="/img/marketplace/marketplace-3.png" alt="MCP 安装模态框显示安装范围和方法选项" width="500" />

对于 MCP，您可能还需要：
- 选择安装方法（NPX 或 Docker）
- 在出现提示时提供其他参数（请参阅[参数化 MCP](#parameterized-mcps)）

#### 安装模式
<img src="/img/marketplace/marketplace-5.png" alt="模式安装模态框显示安装范围选项" width="500" />

对于模式，只需选择安装范围并单击“安装”。

5. Roo Code 会自动将项目添加到相应的配置文件中。如果文件不存在，Roo Code 会为您创建它。然后文件会打开供您审阅。

### 移除项目

1. 在市场视图中找到已安装的项目（已安装的项目会显示“移除”按钮）。
2. 点击“移除”按钮。
3. 如果项目同时安装在两个范围中，请选择是从当前项目移除还是全局移除。
4. Roo Code 会从相应的配置文件中移除该项目。

**注意：** “移除”按钮是上下文感知的。如果一个项目只安装在一个范围中（例如，仅为项目安装），它将是一个单一操作按钮。只有当项目同时安装在两个范围中时，才会出现带有“从项目移除”和“全局移除”选项的下拉菜单。

**重要提示：** 在您单击按钮或从下拉菜单中选择一个选项后，移除操作会立即执行。没有额外的确认提示。

### 参数化 MCP

<img src="/img/marketplace/marketplace-6.png" alt="参数化 MCP 安装显示 API 密钥输入和先决条件" width="500" />

一些 MCP 在安装过程中需要特定信息，例如 API 密钥或 URL。在安装这些“参数化”MCP 时，系统会提示您：
- 查看任何先决条件（如创建帐户或获取 API 密钥）
- 输入所需的配置值
- 如果适用，选择安装方法

这样可以保护敏感信息的安全，并使配置更加灵活。

## 故障排除

### 安装错误
- **无效的 YAML/JSON**: 配置文件（`.roomodes`、`.roo/mcp.json` 等）存在语法错误。为防止数据丢失，Roo Code 不会修改损坏的配置文件。请在安装或移除项目前修复语法错误。
- **文件未找到**: 罕见错误 - Roo Code 会自动创建必要的配置文件。

### 项目不工作
如果已安装的项目不工作：
1. **检查配置文件**: 验证项目是否已正确添加。
2. **重启 VS Code**: 新配置有时需要重启才能生效。
3. **检查先决条件**: 查看项目描述中的任何要求。
4. **检查 Roo Code 日志**: 在 Roo Code 输出面板中查找错误消息。
